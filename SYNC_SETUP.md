# 云端同步功能配置指南

## 功能说明

甜甜育儿记录本支持通过 Supabase 实现云端数据同步，让家人可以共享同一份记录数据。

**主要特性：**
- 📱 多设备同步，家人共享数据
- 💾 本地缓存，离线也能记录
- 🔄 联网后自动同步
- 👨‍👩‍👧 通过家庭码实现数据隔离

## 配置步骤

### 第一步：创建 Supabase 项目

1. 访问 [supabase.com](https://supabase.com) 并注册账号
2. 点击 "New Project" 创建新项目
3. 填写项目信息：
   - Name: 项目名称（如：甜甜育儿记录）
   - Database Password: 设置数据库密码
   - Region: 选择离你最近的区域
4. 等待项目创建完成（约2分钟）

### 第二步：获取项目配置信息

1. 进入项目后台，点击左侧菜单的 "Settings" → "API"
2. 复制以下信息：
   - **Project URL**: 项目URL（如：`https://xxx.supabase.co`）
   - **anon public key**: 公开密钥

### 第三步：创建数据库表

1. 进入项目后台，点击左侧菜单的 "SQL Editor"
2. 点击 "New query" 创建新查询
3. 打开项目中的 `supabase-schema.sql` 文件
4. 将文件中的所有SQL代码复制到查询编辑器中
5. 点击 "Run" 执行SQL，创建所有数据表

### 第四步：在应用中配置

1. 打开甜甜育儿记录本
2. 进入「设置」页面
3. 找到「云端同步」区域
4. 填写以下信息：
   - **Supabase URL**: 第二步复制的Project URL
   - **anon key**: 第二步复制的anon public key
   - **家庭码**: 设置一个家庭共享码（如：`tiantian2026`）
5. 点击「启用同步」

### 第五步：家人共享

家人使用相同的配置即可共享数据：
1. 在家人的手机/电脑上打开应用
2. 进入设置 → 云端同步
3. 填写相同的 Supabase URL、anon key 和 家庭码
4. 点击启用同步，即可看到相同的记录

## 使用说明

### 同步机制
- **手动同步**: 点击「立即同步」按钮手动触发
- **自动同步**: 每5分钟自动检查并同步待处理的记录
- **离线使用**: 没有网络时数据保存在本地，联网后自动同步

### 家庭码说明
- 家庭码用于区分不同的家庭数据
- 同一个家庭码的用户共享同一份数据
- 建议设置一个不容易被猜到的家庭码
- 可以随时修改家庭码切换到不同的数据

### 数据安全
- anon key 只能访问公开数据，请确保已正确配置RLS策略
- 家庭码作为简单的数据隔离方式，不适合高度敏感数据
- 建议定期导出数据备份

## 常见问题

**Q: 配置后看不到数据？**
A: 检查家庭码是否正确，确认数据库表已创建成功。

**Q: 同步失败怎么办？**
A: 检查网络连接，确认Supabase URL和anon key正确。

**Q: 可以多个家庭使用同一个项目吗？**
A: 可以，通过不同的家庭码实现数据隔离。

**Q: 数据会冲突吗？**
A: 目前是简单的最后写入为准策略，建议家人错开时间编辑。

## 技术细节

### 数据表结构
- `families`: 家庭信息
- `mother_cats`: 母猫信息
- `kittens`: 幼崽信息
- `feeding_logs`: 喂食记录
- `medicine_logs`: 喂药记录
- `nursing_logs`: 吃奶记录
- `photos`: 照片
- `kitten_records`: 成长记录
- `reminders`: 提醒
- `settings`: 设置

### 本地存储
- Supabase配置: `tiantian_supabase_config`
- 同步队列: `tiantian_sync_queue`
- 最后同步时间: `tiantian_last_sync_time`
- 家庭信息: `tiantian_family_info`
